## Unsatisfied Requirements

### FTTL-04: WebSocket toast notification when file auto-deleted

**Requirement:** Connected browsers receive a WebSocket toast notification when a file is auto-deleted.

**Break:** `run_file_ttl_sweep` is called with `broadcast_fn=None` at `relay/app/main.py:74`. The broadcast block in `file_ttl_sweep.py:48` is guarded by `if broadcast_fn is not None` — so no toast is ever sent. Additionally, the drop box WebSocket at `mount_proxy.py:279` immediately closes browser WS connections, so no browser can maintain a WS channel to the drop box.

**Fix needed:**
1. Wire a broadcast function from the drop box's connection manager into `run_file_ttl_sweep`
2. Keep drop box browser WS connections alive (don't close at mount_proxy.py:279)

### FTTL-06: On mount restart, user prompted to keep or delete expired files

**Requirement:** On mount restart, the user is prompted whether to keep or delete files with expired TTLs.

**Break:** The relay correctly sends `expired_files` control message to the agent on reclaim (agent_ws.py:186-196). The agent handles it and prompts the user (agent/connection.py:135-170). The agent sends back `delete_expired_files` or `keep_expired_files`. But the relay's `TunnelConnection.run_receive_loop` at `tunnel/connection.py:276-293` only handles `pong` and `ping` — the agent's response is silently dropped at line 289.

**Fix needed:** Add handler branches in `tunnel/connection.py` `run_receive_loop` for `delete_expired_files` and `keep_expired_files` message types, forwarding to file TTL DB cleanup logic.
