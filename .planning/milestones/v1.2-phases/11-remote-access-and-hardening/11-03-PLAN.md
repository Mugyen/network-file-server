<objective>
Add WebSocket tunneling so real-time features (clipboard sync, transfer notifications, device discovery) work through the relay proxy.

Purpose: Browser opens WS to `/m/{code}/ws`, relay bridges frames bidirectionally through the tunnel to the agent, which opens a local WS to the ASGI app. This gives full LAN-mode parity for all real-time features in remote mode.

Output: Three new FrameType values (WS_OPEN, WS_DATA, WS_CLOSE), relay WS endpoint, agent WS bridge handler, httpx-ws promoted to production dependency.
</objective>
