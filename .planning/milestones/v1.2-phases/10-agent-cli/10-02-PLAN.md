    **Create `agent/connection.py`:**

    `connect_and_serve(relay_url: str, folder: Path, name: str, preferred_code: str | None) -> str`:
    - Build ws_url: `{relay_url.rstrip('/')}/agent/ws` with `?code={preferred_code}` if provided
    - Use `websockets.asyncio.client.connect(ws_url, ping_interval=None)` — disable built-in ping (TunnelConnection handles heartbeat)
    - Wrap raw WS in `WebSocketClientAdapter`, then `TunnelConnection(adapter)`
    - `await conn.receive_control()` — expect `{"type": "mount_registered", "code": "..."}`; raise ValueError if wrong type
    - Start heartbeat: `conn.start_heartbeat(HEARTBEAT_INTERVAL_S, HEARTBEAT_MISSED_LIMIT)`
    - Call `print_mounted(relay_url, assigned_code, folder, name)`
    - Create `ServerConfig(shared_folder=folder, port=0, password_hash=None, read_only=False, receive=False)` and `set_server_config(config)`
    - Create `app = create_app()`, `transport = ASGITransport(app=app)`, `AsyncClient(transport=transport, base_url="http://local")`
    - For receiving and dispatching OPEN frames: The agent needs to intercept OPEN frames from run_receive_loop. Since TunnelConnection._dispatch_frame handles DATA/CLOSE/CANCEL by routing to streams, and OPEN frames are not currently dispatched (they cause FrameType lookup to fail or be ignored), the agent must implement its own receive loop that reads raw frames, dispatches OPEN to handle_open_frame via create_task, and routes other binary frame types through conn._dispatch_frame. Text frames (control messages like pong) are handled by calling conn.handle_pong() for pong type.
    - Implement `_agent_receive_loop(conn: TunnelConnection, asgi_client: AsyncClient) -> None` that:
      1. Calls `await conn._ws.receive()` in a loop
      2. For `{"bytes": raw}`: deserialize_frame -> if OPEN, create task for handle_open_frame; else conn._dispatch_frame
      3. For `{"text": text}`: parse JSON, if type=="pong" call conn.handle_pong()
      4. Track pending tasks in a set, clean up on exit
    - Return `assigned_code` so reconnect loop can send it as preferred

    `run_agent_loop(relay_url: str, folder: Path, name: str) -> None`:
    - Outer while-True loop with try/except around `connect_and_serve`
    - On success (clean disconnect): reset attempt=0
    - On exception: increment attempt, compute delay via `compute_backoff(attempt, 1.0, 60.0, 0.5)`, call `print_reconnect_status`, `await asyncio.sleep(delay)`
    - Track `last_code` across iterations for preferred code on reconnect
    - On `KeyboardInterrupt`: break (handled by caller)
