  <behavior>
    Agent WebSocket:
    - Agent connects to /agent/ws?code=XXXX, relay accepts, wraps in TunnelConnection, registers in registry, starts heartbeat, runs receive loop
    - On WebSocket disconnect, mount is deregistered and TunnelConnection is closed
    - Deregister in finally block is wrapped in try/except MountNotFoundError (heartbeat may have already cleaned up)

    HTTP Proxy:
    - GET /m/{code}/some/path proxies request through TunnelConnection and returns StreamingResponse with body from agent
    - POST /m/{code}/upload with body forwards body bytes in OPEN frame metadata
    - All HTTP methods (GET, POST, PUT, DELETE, PATCH) are handled by single api_route
    - OPEN frame metadata includes: method, path (prefixed with /), query string, filtered headers (hop-by-hop stripped), body (latin-1 encoded)
    - Response metadata (status, headers) read from first stream chunk or control message before starting StreamingResponse
    - Hop-by-hop headers (Connection, Keep-Alive, TE, Trailers, Transfer-Encoding, Upgrade, Proxy-Authenticate, Proxy-Authorization) are stripped
    - When browser disconnects mid-stream, CANCEL frame sent to agent
    - When mount not found: 404 with not_found.html (which has code input per RELY-04)
    - When mount offline: 503 with offline.html
    - When mount expired: 410 with expired.html
    - FirstByteTimeoutError returns 504 Gateway Timeout
  </behavior>

       - `open_stream(request_id)` records the request_id
       - `send_open(request_id, metadata)` stores metadata for assertion
       - `read_stream(request_id, timeout_s)` returns a pre-configured first chunk (response metadata JSON)
       - `read_stream_iter(request_id)` yields pre-configured body chunks
       - `send_cancel(request_id)` records cancellation for assertion
       - `close()` no-op async
       - `start_heartbeat(interval, limit)` no-op
       - `run_receive_loop()` awaits an asyncio.Event that tests can set to simulate disconnect

       - Add `mock_connection` fixture that returns a configured MockTunnelConnection
       - Add `registered_relay_client` fixture that pre-registers a mock connection under code "testcode" in the registry

    2. **Create `relay/app/routers/agent_ws.py`**:
       - `router = APIRouter()`
       - `@router.websocket("/agent/ws")` handler accepting `websocket: WebSocket` and `code: str = Query(...)`
       - Accept WebSocket, wrap in `TunnelConnection(websocket)`, register in `get_registry()` with the provided code
       - Call `conn.start_heartbeat(HEARTBEAT_INTERVAL_S, HEARTBEAT_MISSED_LIMIT)`
       - `await conn.run_receive_loop()` in try block
       - Catch `WebSocketDisconnect` (pass)
       - Finally: `await conn.close()`, then `try: get_registry().deregister(code) except MountNotFoundError: pass`

    3. **Create `relay/app/routers/mount_proxy.py`**:
       - `router = APIRouter()`
       - Import `templates` from `relay.app.routers.landing` for error page rendering
       - Define `HOP_BY_HOP = frozenset({...})` constant with standard hop-by-hop headers (all lowercase)
       - `@router.api_route("/m/{code}/{path:path}", methods=["GET", "POST", "PUT", "DELETE", "PATCH"])` handler
       - Return type annotation: `Response` (base class — both TemplateResponse and StreamingResponse are subtypes)
       - Lookup connection via `get_registry().get_connection(code)` — catch MountNotFoundError (404 not_found.html), MountOfflineError (503 offline.html), MountExpiredError (410 expired.html). Return `TemplateResponse` with `status_code` kwarg.
       - Read full request body upfront: `body = await request.body()`
       - Build forward headers: filter out hop-by-hop, rewrite Host
       - Build metadata dict: `{"method": request.method, "path": f"/{path}", "query": str(request.url.query), "headers": forward_headers, "body": body.decode("latin-1") if body else ""}`
       - Generate `request_id = uuid.uuid4()`, call `conn.open_stream(request_id)`, then `await conn.send_open(request_id, metadata)`
       - Read first chunk via `await conn.read_stream(request_id, FIRST_BYTE_TIMEOUT_S)` — this is the response metadata JSON (status code + response headers). Parse it to extract status and response headers.
       - Define `async def stream_generator()` that yields body chunks from `conn.read_stream_iter(request_id)`, checking `await request.is_disconnected()` between chunks and sending CANCEL if disconnected
       - Return `StreamingResponse(stream_generator(), status_code=resp_status, headers=resp_headers, media_type=resp_content_type)`
       - Catch `FirstByteTimeoutError` and return `Response("Gateway Timeout", status_code=504)`
       - Also add a bare `/m/{code}` route (no trailing path) that redirects to `/m/{code}/` for convenience, OR handle `path=""` as the root

    4. **Update `relay/app/main.py`** — Include agent_ws router and mount_proxy router via `app.include_router()`.

       - `test_proxy_not_found` — GET /m/unknown/path, assert 404, assert "Mount Not Found" in response HTML, assert code input form present
       - `test_proxy_offline` — Register mount then mark_offline, GET /m/testcode/path, assert 503, assert "offline" in response HTML
       - `test_proxy_expired_page` — Register mount with EXPIRED status, GET /m/testcode/path, assert 410, assert "expired" in response HTML
