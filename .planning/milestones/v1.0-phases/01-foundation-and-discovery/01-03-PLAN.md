Backend API endpoints:

GET /api/files?path={relative_path}
Response (DirectoryListing):
```json
{
  "path": "",
  "entries": [
    {
      "name": "document.pdf",
      "size": 1048576,
      "size_display": "1.0 MB",
      "type": "file",
      "modified": "2026-03-09T10:30:00"
    },
    {
      "name": "photos",
      "size": 0,
      "size_display": "0 B",
      "type": "directory",
      "modified": "2026-03-08T15:00:00"
    }
  ]
}
```

GET /api/server-info
Response (ServerInfo):
```json
{
  "ip": "192.168.1.5",
  "port": 8000,
  "url": "http://192.168.1.5:8000",
  "qr_svg": "<svg xmlns=\"http://www.w3.org/2000/svg\" ...>...</svg>",
  "all_ips": ["192.168.1.5", "10.0.0.2"]
}
```

Error responses:
- 403: {"detail": "Path traversal blocked: ..."}
- 404: {"detail": "Path not found: ..."}

FileType enum values: "file", "directory"

       - API_BASE constant set to "/api"
       - ApiError class extending Error with status (number) and body (string) properties

    8. Create client/src/api/files.ts:
       - fetchFiles(path: string): Promise<DirectoryListing> -- calls apiFetch with /files?path={encodeURIComponent(path)}

    9. Create client/src/api/serverInfo.ts:
       - fetchServerInfo(): Promise<ServerInfo> -- calls apiFetch with /server-info

    10. Verify the project builds: `cd client && npx tsc --noEmit` (type check) and `npm run build` (production build).
