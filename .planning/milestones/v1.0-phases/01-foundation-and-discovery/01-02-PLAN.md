  <behavior>
    - generate_ascii_qr("http://192.168.1.5:8000") returns a non-empty string containing ASCII characters
    - generate_ascii_qr output is multi-line (contains newlines)
    - generate_svg_qr("http://192.168.1.5:8000") returns a string starting with "<?xml" or "<svg" (valid SVG)
    - generate_svg_qr output contains "path" element (SVG path data)
    - Both QR functions raise ValueError if url is empty string
    - detect_primary_lan_ip() returns a string matching IPv4 pattern (x.x.x.x) and not starting with "127."
    - detect_all_lan_ips() returns a non-empty list of IPv4 strings, none starting with "127."
  </behavior>
  <action>
    1. Create server/app/services/qr_service.py:
       - generate_ascii_qr(url: str) -> str: validates url is non-empty (raise ValueError), creates qrcode.QRCode(), adds data, makes it, uses print_ascii() with StringIO buffer, returns the string.
       - generate_svg_qr(url: str) -> str: validates url is non-empty (raise ValueError), uses qrcode.make() with image_factory=qrcode.image.svg.SvgPathImage, saves to BytesIO buffer, returns decoded UTF-8 string.

    2. Create server/app/services/network_service.py:
       - detect_primary_lan_ip() -> str: uses socket UDP trick (connect to 8.8.8.8:80, read getsockname). Raises RuntimeError if OSError occurs (no network). Does NOT fall back silently to 127.0.0.1 (per project convention: throw exceptions, don't return fallback values).
       - detect_all_lan_ips() -> list[str]: uses ifaddr.get_adapters() to enumerate all non-loopback IPv4 addresses. Raises RuntimeError if no addresses found.

    3. Create server/tests/test_qr_service.py:
       - TestAsciiQr: test output is non-empty multi-line string, test empty URL raises ValueError
       - TestSvgQr: test output contains SVG markup, test empty URL raises ValueError

    4. Create server/tests/test_network.py:
       - test detect_primary_lan_ip returns valid IPv4 (not 127.x)
       - test detect_all_lan_ips returns non-empty list of valid IPv4 addresses
       - Note: these tests call real network APIs so they verify actual behavior on the dev machine. If CI has no network, mock the socket; for now test on real machine.

    5. Run: `uv run pytest server/tests/test_qr_service.py server/tests/test_network.py -x -v`
