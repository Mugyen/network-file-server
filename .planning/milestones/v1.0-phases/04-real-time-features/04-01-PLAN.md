     - `type WSMessageType = (typeof WSMessageType)[keyof typeof WSMessageType]`
     - `ToastType` as const: FILE_UPLOADED, DEVICE_CONNECTED, DEVICE_DISCONNECTED, REQUEST_CREATED, REQUEST_FULFILLED.
     - `type ToastType = (typeof ToastType)[keyof typeof ToastType]`
     - `interface ToastMessage { id: string; toastType: ToastType; message: string; deviceName: string; timestamp: string; }`
     - `interface WSToastPayload { type: typeof WSMessageType.TOAST; toast_type: ToastType; message: string; device_name: string; timestamp: string; }`
     - `interface WSDeviceCountPayload { type: typeof WSMessageType.DEVICE_COUNT; count: number; }`
     - `type WSMessage = WSToastPayload | WSDeviceCountPayload | { type: string; [key: string]: unknown; }` (extensible union for future message types from Plans 02/03).
     - Device name generation: `generateDeviceName()` function picking random adjective + animal from small arrays (~30 each). Store/retrieve from localStorage key `wfs_device_name`.
     - `getDeviceName()`: Read from localStorage, if absent call generateDeviceName and store.

  3. **Create `client/src/hooks/useWebSocket.ts`**:
     - Accepts `deviceName: string` parameter.
     - Returns `{ isConnected: boolean; deviceCount: number; sendMessage: (msg: object) => void; addMessageHandler: (type: string, handler: (data: unknown) => void) => void; removeMessageHandler: (type: string) => void; }`.
     - Store WebSocket instance in `useRef<WebSocket | null>` (NOT useState -- avoids stale closure issues per research pitfall 1).
     - Store message handlers in `useRef<Map<string, (data: unknown) => void>>`.
     - On mount: construct URL `ws://${window.location.host}/ws?device_name=${encodeURIComponent(deviceName)}`. Create WebSocket.
     - `onopen`: set isConnected=true, reset attempt counter to 0.
     - `onmessage`: parse JSON, if type==="device_count" update deviceCount state. Otherwise look up handler in handlersRef by message type and call it.
     - `onclose`: set isConnected=false. Schedule reconnect via `setTimeout` with exponential backoff: `min(1000 * 2^attempt, 30000) + random(0,1000)` jitter. Increment attempt counter.
     - `sendMessage(msg)`: call `wsRef.current?.send(JSON.stringify(msg))`. No-op if not connected.
     - `addMessageHandler(type, handler)` / `removeMessageHandler(type)`: modify handlersRef.current Map.
     - Cleanup on unmount: close WebSocket, clear reconnect timeout.

  4. **Create `client/src/hooks/useToast.ts`**:
     - Returns `{ toasts: ToastMessage[]; addToast: (payload: WSToastPayload) => void; dismissToast: (id: string) => void; }`.
     - State: `toasts` array of ToastMessage.
     - `addToast`: create ToastMessage with unique id (Date.now + random), add to state. If count > 3, only keep latest 3 (older ones are tracked for "+N more" count). Set setTimeout(4000) for auto-dismiss. Store timeout id in a `useRef<Map<string, number>>` so manual dismiss can clearTimeout (prevents memory leak per research pitfall 6).
     - `dismissToast(id)`: remove from state, clearTimeout for that toast.
     - Compute `overflowCount` = total toasts beyond visible 3.

  5. **Create `client/src/components/Toast.tsx`**:

Slide-in animation via Tailwind: `animate-[slideInRight_0.3s_ease-out]` with a `@keyframes slideInRight` defined in index.css (`from { transform: translateX(100%); opacity: 0; } to { transform: translateX(0); opacity: 1; }`).

  6. **Create `client/src/components/ToastContainer.tsx`**:

  7. **Create `client/src/components/ConnectionStatus.tsx`**:

  8. **Wire into `client/src/App.tsx`**:
