     - `ClipboardService` class:
       - Constructor takes `data_dir: Path`. Computes `file_path = data_dir / "clipboard.json"`. Creates data_dir if not exists.
       - `_lock: asyncio.Lock` for safe concurrent read-modify-write (per research pitfall 4).
       - `_snippets: dict[str, dict]` in-memory cache loaded from file on first access.
       - `async def _ensure_loaded(self) -> None`: Load from file if not yet loaded.
       - `async def list_snippets(self) -> list[Snippet]`: Return all snippets sorted by created_at.
       - `async def create_snippet(self, title: str) -> Snippet`: Generate UUID id, set empty content, timestamps. Validate count < 50, raise ValueError if exceeded. Persist. Return Snippet.
       - `async def update_snippet(self, snippet_id: str, content: str) -> Snippet`: Validate content length <= 10000, raise ValueError if exceeded. Find snippet, raise KeyError if not found. Update content + updated_at. Persist. Return updated Snippet.
       - `async def update_title(self, snippet_id: str, title: str) -> Snippet`: Find snippet, raise KeyError if not found. Update title + updated_at. Persist. Return updated Snippet.
       - `async def delete_snippet(self, snippet_id: str) -> None`: Find snippet, raise KeyError if not found. Remove. Persist.
       - `async def _persist(self) -> None`: Acquire lock, call write_json_atomic with snippets dict.
     - Module-level factory: `def get_clipboard_service() -> ClipboardService` that lazily creates singleton using `get_server_config().shared_folder.parent / ".wfs_data"` as data_dir.

  3. **Create `server/app/routers/clipboard.py`**:
     - `router = APIRouter(prefix="/api/clipboard", tags=["clipboard"])`
     - `GET /` -> list[Snippet]: Return all snippets.
     - `POST /` -> Snippet: Accept CreateSnippetRequest, create snippet, broadcast SNIPPET_CREATED via WS manager to all connections, return snippet.
     - `PATCH /{snippet_id}` -> Snippet: Accept UpdateSnippetTitleRequest, update title, broadcast SNIPPET_UPDATED, return snippet. (Title updates come via REST, not WS, because they are less frequent.)
     - `DELETE /{snippet_id}` -> dict: Delete snippet, broadcast SNIPPET_DELETED via WS manager to all connections, return `{"status": "deleted"}`.

  4. **Add clipboard WS message handler** in `server/app/routers/websocket.py`:

This is the real-time sync path for content edits (debounced from client).

  5. **Register clipboard router** in `server/app/main.py`:

  1. **Create `client/src/types/clipboard.ts`**:
     - `interface Snippet { id: string; title: string; content: string; created_at: string; updated_at: string; }`

  2. **Create `client/src/api/clipboard.ts`**:
     - `fetchSnippets(): Promise<Snippet[]>` -> GET /api/clipboard
     - `createSnippet(title: string): Promise<Snippet>` -> POST /api/clipboard
     - `updateSnippetTitle(snippetId: string, title: string): Promise<Snippet>` -> PATCH /api/clipboard/{id}
     - `deleteSnippet(snippetId: string): Promise<{ status: string }>` -> DELETE /api/clipboard/{id}
     All using apiFetch/apiPost/apiPatch/apiDelete from client.ts.

  3. **Create `client/src/hooks/useClipboard.ts`**:
     - Parameters: `addMessageHandler`, `removeMessageHandler`, `sendMessage` from useWebSocket (passed in, not calling useWebSocket directly -- avoids duplicate hook instantiation).
     - State: `snippets: Snippet[]`, `isOpen: boolean` (panel visibility), `isLoading: boolean`.
     - On mount: fetch all snippets via REST (fetchSnippets). Register WS handlers for SNIPPET_CREATED (add to state), SNIPPET_UPDATED (replace in state), SNIPPET_DELETED (remove from state).
     - `togglePanel()`: flip isOpen.
     - `addSnippet()`: call createSnippet("Untitled") via REST. Server broadcasts to other clients. Add to local state immediately.
     - `updateContent(snippetId: string, content: string)`: Update local state immediately (optimistic). Send WS message `{ type: "snippet_update", snippet_id: snippetId, content }`. Debounce the WS send by 300ms using a `useRef<Map<string, number>>` of timeout IDs per snippet (clear previous timeout on each keystroke, set new 300ms timeout). This gives instant local feedback while rate-limiting network traffic.
     - `updateTitle(snippetId: string, title: string)`: Call updateSnippetTitle via REST (no WS debounce needed for infrequent title edits). Update local state.
     - `removeSnippet(snippetId: string)`: Call deleteSnippet via REST. Server broadcasts deletion to other clients. Remove from local state.
     - Cleanup: remove all WS handlers on unmount, clear all debounce timeouts.
     - Returns: `{ snippets, isOpen, isLoading, togglePanel, addSnippet, updateContent, updateTitle, removeSnippet }`.

  4. **Create `client/src/components/SnippetCard.tsx`**:
     - Props: `snippet: Snippet`, `onUpdateContent: (id: string, content: string) => void`, `onUpdateTitle: (id: string, title: string) => void`, `onDelete: (id: string) => void`.
     - State: `isCollapsed: boolean` (default false).
     - Renders:
       - Header bar with: editable title input (blur/Enter saves via onUpdateTitle), collapse/expand chevron button (ChevronDown/ChevronRight from lucide-react), delete X button.
