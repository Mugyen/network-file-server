<objective>
File request system where users can ask for specific files and others can fulfill the requests.

Purpose: Enable collaborative file sharing where one device can request a specific file and another device on the LAN can upload it directly, with real-time status updates across all connected devices.

Output: File request creation form, banner display above file list, fulfillment via upload/drag-drop, and real-time status sync.
</objective>

Broadcast REQUEST_CREATED via WS to all OTHER connections. Also broadcast a toast with toast_type=REQUEST_CREATED to all OTHER connections. Return request.

     - `DELETE /{request_id}` -> dict: Requires `X-Device-Id` header. Dismiss request (validates requester). Broadcast REQUEST_DISMISSED to all connections. Return `{"status": "dismissed"}`.
