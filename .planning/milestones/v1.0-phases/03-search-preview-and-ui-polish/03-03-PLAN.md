**1. Create `client/src/hooks/usePreview.ts`:**

```typescript
interface PreviewState {
  previewFile: FileEntry | null;
  openPreview: (file: FileEntry) => void;
  closePreview: () => void;
  textContent: string | null;
  isLoadingContent: boolean;
  contentError: string | null;
}
export function usePreview(currentPath: string): PreviewState;
```

Implementation:
- `previewFile` state manages which file is being previewed (null = closed)
- `openPreview(file)`: set previewFile. If file category is CODE, TEXT, or MARKDOWN, fetch the file content as text from `/api/files/preview?path={fullPath}` (use `fetch` directly, read `response.text()`). Set `isLoadingContent` during fetch. Store in `textContent`. Apply a 500KB size limit check -- if `Content-Length` header exceeds 500KB, set `contentError` to "File too large to preview" and skip fetch.
- `closePreview()`: set previewFile to null, clear textContent, clear contentError
- When `currentPath` changes, close preview
- Build full path from currentPath + file.name (same pattern as FileRow's `buildChildPath`)

```typescript
interface PreviewModalProps {

  files: FileEntry[];       // currently visible file list (for gallery navigation)

  textContent: string | null;

  contentError: string | null;

}
```

Implementation:

**Shell/chrome:**
- Fixed overlay: `fixed inset-0 z-60 bg-black/50 flex items-center justify-center` (same pattern as ConfirmDialog but z-60 to be above other modals)
- Modal card: `bg-white dark:bg-gray-800 rounded-lg shadow-xl max-w-5xl w-full mx-4 max-h-[90vh] flex flex-col`
- Header bar: file name (truncated with ellipsis), "Open in new tab" button (ExternalLink icon, opens `/api/files/preview?path=...` in new tab via `window.open`), download button (Download icon, links to `/api/files/download?path=...`), close button (X icon)
- Body: scrollable content area

**Keyboard handling:**
- Escape key closes modal (add keydown listener in useEffect)
- Click on backdrop (outside modal card) closes modal
- ArrowLeft/ArrowRight delegated to ImagePreview when showing images

**Content switching by category:**
- Use `getFileCategory(file.name)` to determine which preview component to render
- Build `previewUrl` = `/api/files/preview?path=${encodeURIComponent(fullPath)}` where fullPath = currentPath ? `${currentPath}/${file.name}` : file.name
- Build `downloadUrl` = `/api/files/download?path=${encodeURIComponent(fullPath)}`

Switch logic:
- `FileCategory.IMAGES`: Render ImagePreview. Build `imageFiles` list by filtering `files` for image category. Compute `currentIndex` in that list. `onNavigate` calls `onNavigateFile` with the new image FileEntry.
- `FileCategory.VIDEO`: Render VideoPreview with previewUrl.
- `FileCategory.AUDIO`: Render AudioPreview with previewUrl.
- `FileCategory.DOCUMENTS`: If file extension is "pdf", render PdfPreview with previewUrl. Otherwise render FileInfoPreview.
- `FileCategory.CODE`: If `isLoadingContent`, show a loading spinner. If `contentError`, show error message. Otherwise render CodePreview with `textContent` and `isDark`.
- `FileCategory.MARKDOWN`: Same loading/error pattern, render MarkdownPreview with `textContent`.
- `FileCategory.TEXT`: Same loading/error pattern, render CodePreview with language="text" (plain text with line numbers, no highlighting).
- Everything else (ARCHIVES, EXECUTABLES, ALL): Render FileInfoPreview with download button.
