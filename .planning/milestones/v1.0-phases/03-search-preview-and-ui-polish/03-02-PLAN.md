**2. Create `client/src/hooks/useSearch.ts`:**

Manages search query, debounced backend calls, and client-side filtering.

```typescript
interface SearchState {
  query: string;
  setQuery: (q: string) => void;
  isSearching: boolean;
  searchResults: FileEntry[] | null;  // null = not searching, [] = no results
  filterFiles: (files: FileEntry[]) => FileEntry[];
}
export function useSearch(currentPath: string): SearchState;
```

Implementation:
- `query` state (string), updated by `setQuery`
- When query is non-empty, apply 300ms debounce (useRef for timeout ID) then call `searchFiles(query, currentPath)`. Set `isSearching` true during fetch, store results in `searchResults`.
- `filterFiles(files)`: if query is non-empty but searchResults is null (still debouncing), filter files client-side by `file.name.toLowerCase().includes(query.toLowerCase())`. If searchResults is not null, return searchResults directly (backend recursive results).
- When query becomes empty, set `searchResults` to null, clear timeout.
- When `currentPath` changes, clear query and searchResults.
- Clean up timeout on unmount.

**3. Create `client/src/hooks/useSort.ts`:**

```typescript
export const SortField = { NAME: "name", SIZE: "size", MODIFIED: "modified" } as const;
export type SortField = (typeof SortField)[keyof typeof SortField];
export const SortDirection = { ASC: "asc", DESC: "desc" } as const;
export type SortDirection = (typeof SortDirection)[keyof typeof SortDirection];

interface SortState {
  field: SortField;
  direction: SortDirection;
  toggleSort: (field: SortField) => void;
  sortFiles: (files: FileEntry[]) => FileEntry[];
}
export function useSort(): SortState;
```

Implementation:
- Default state: `field: SortField.NAME, direction: SortDirection.ASC`
- `toggleSort(clickedField)`: if same field, flip direction; if different field, set new field with ASC
- `sortFiles(files)`: returns a new sorted array. CRITICAL: always sort directories before files first, then apply user sort within each group. Use `FileType.DIRECTORY` check. For `NAME`: locale-aware `localeCompare`. For `SIZE`: numeric comparison. For `MODIFIED`: string comparison (ISO dates sort lexicographically).
- Use `useMemo` for the comparator to avoid recalculating on every render.

**4. Create `client/src/hooks/useTheme.ts`:**

```typescript
export const ThemeMode = { LIGHT: "light", DARK: "dark", SYSTEM: "system" } as const;
export type ThemeMode = (typeof ThemeMode)[keyof typeof ThemeMode];

interface ThemeState {
  mode: ThemeMode;
  isDark: boolean;
  setMode: (mode: ThemeMode) => void;
}
export function useTheme(): ThemeState;
```

Implementation per RESEARCH.md Pattern 4:
- Read initial mode from `localStorage.getItem("theme")`. If null or not "light"/"dark", default to `ThemeMode.SYSTEM`.
- `isDark`: if LIGHT return false, if DARK return true, if SYSTEM check `matchMedia("(prefers-color-scheme: dark)").matches`
- `useEffect` syncs `.dark` class on `document.documentElement` when `isDark` changes
- `useEffect` listens for system preference changes when mode is SYSTEM (add/remove `matchMedia` change listener)
- `setMode`: update state, persist to localStorage (remove key for SYSTEM mode)
- Force re-render on system preference change by using useState for the media query result
