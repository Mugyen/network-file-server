**1. Create `client/src/types/fileCategories.ts`** -- the shared file category system:

Define `FileCategory` as an `as const` object with values: `"all"`, `"images"`, `"video"`, `"audio"`, `"documents"`, `"text"`, `"code"`, `"markdown"`, `"archives"`, `"executables"`. Export the type union.

Create `EXTENSION_CATEGORY_MAP: Record<string, FileCategory>` mapping every extension from CONTEXT.md to its category:
- Images: jpg, jpeg, png, gif, svg, webp, bmp, ico, tiff
- Video: mp4, webm, mov, avi, mkv, flv, wmv
- Audio: mp3, wav, ogg, flac, aac, m4a, wma
- Documents: pdf, doc, docx, xls, xlsx, ppt, pptx, odt, ods, odp
- Text: txt, csv, log, ini, cfg, conf, env
- Code: js, ts, jsx, tsx, py, go, rs, java, c, cpp, h, hpp, rb, php, swift, kt, scala, sh, bash, zsh, sql, html, css, scss, yaml, yml, json, xml, toml
- Markdown: md, mdx
- Archives: zip, tar, gz, bz2, xz, rar, 7z, tgz
- Executables: exe, msi, dmg, app, bin, deb, rpm, apk, elf, out

Export `getFileCategory(fileName: string): FileCategory` -- extracts extension, looks up in map, returns `FileCategory.ALL` if not found. Throw if fileName is empty string.

Export `isPreviewable(fileName: string): boolean` -- returns true if category is images, video, audio, documents, text, code, or markdown (not archives, executables, or ALL).

Export `getCategoryExtensions(category: FileCategory): string[]` -- returns all extensions for a given category.

Export `CATEGORY_METADATA: Record<FileCategory, { label: string; icon: string }>` with display labels ("All", "Images", "Video", etc.) and Lucide icon names ("layers", "image", "video", "music", "file-text", "type", "code", "book-open", "archive", "cpu").

**2. Add `search_files()` to `server/app/services/file_service.py`:**

```python
def search_files(base_dir: Path, search_root: str, query: str) -> list[FileEntry]:
```

Walk directory tree from `search_root` using `Path.rglob("*")`. For each item:
- Validate with `is_relative_to(base_resolved)` to prevent symlink escape
- Match `query.lower()` against `item.name.lower()` (contains match)
- Build relative path from search_root (not base_dir) for the `name` field so results show paths like `subdir/file.txt`
- Create FileEntry with `format_file_size`, `FileType`, ISO timestamp
- Skip items that are not files or directories (e.g., broken symlinks)

Raise `PathTraversalError` if search_root resolves outside base. Raise `ValueError` if query is empty string.

**3. Add `SearchResult` schema to `server/app/models/schemas.py`:**

```python
class SearchResult(BaseModel):
    query: str
    path: str
    entries: list[FileEntry]
```

**4. Add two endpoints to `server/app/routers/files.py`:**

**GET `/api/files/search`** with query params `q` (required) and `path` (optional, default ""):
- Call `search_files(config.shared_folder, path, q)`
- Return `SearchResult(query=q, path=path, entries=results).model_dump()`
- Handle PathTraversalError, FileNotFoundError, ValueError (400 for empty query)

**GET `/api/files/preview`** with query param `path` (required):
- Reuse `download_file(config.shared_folder, path)` to validate and resolve
- Use `mimetypes.guess_type(file_path.name)` for Content-Type (fall back to `application/octet-stream`)
- Return `FileResponse` with `Content-Disposition: inline; filename*=UTF-8''{encoded_name}`
- Import `mimetypes` from stdlib, `quote` from `urllib.parse` (already imported)
- Starlette FileResponse handles Range requests (206 Partial Content) automatically -- no custom code needed
- Handle PathTraversalError, FileNotFoundError, ValueError
