Add connection status overlays to the SPA that replace the current redirect-on-error behavior. When the agent goes offline (503), show a persistent "Host Offline" banner with greyed-out file list. When the mount expires (410), show a "Mount Expired" banner. Add a REST status endpoint for polling.

   ```python
   @router.get("/m/{code}/status")
   async def mount_status(code: str) -> dict[str, str]:
       """Return the current status of a mount code."""
       registry = get_registry()
       try:
           await registry.get_connection(code)
           return {"status": "online"}
       except MountOfflineError:
           return {"status": "offline"}
       except MountExpiredError:
           return {"status": "expired"}
       except MountNotFoundError:
           return {"status": "not_found"}
   ```

   IMPORTANT: This route must NOT have the `@limiter.limit()` decorator. Status polling at 30s intervals should not consume the proxy rate limit budget.
