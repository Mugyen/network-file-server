1. Create `server/app/middleware/__init__.py` (empty).

2. Create `server/app/middleware/auth_middleware.py` -- pure ASGI middleware (NOT BaseHTTPMiddleware, which is deprecated):
   - `class AuthMiddleware` with `__init__(self, app: ASGIApp, token_service: AuthTokenService) -> None`
   - `EXEMPT_PREFIXES = ("/api/auth/login", "/api/server-info", "/assets")` -- tuple for startswith matching
   - `async def __call__(self, scope: Scope, receive: Receive, send: Send) -> None`:
     - If scope["type"] != "http" and scope["type"] != "websocket": pass through (lifespan)
     - If scope["type"] == "websocket": pass through (WebSocket auth checked in endpoint, not middleware -- see research)
     - Check path against EXEMPT_PREFIXES using startswith
     - Parse cookies from scope headers using `http.cookies.SimpleCookie`
     - Get "session" cookie value, validate via token_service.validate_token()
     - If invalid/missing: for API paths (/api/*) return 401 JSON response; for other paths return 401 JSON response (frontend handles redirect)
     - If valid: pass through to app

3. Add `LoginRequest` to `server/app/models/schemas.py`: `password: str` field only. Extend `ServerInfo` with `read_only: bool`, `receive: bool`, `password_required: bool`, `hostname: str`.

4. Create `server/app/routers/auth.py`:
   - `router = APIRouter(prefix="/api/auth", tags=["auth"])`
   - `POST /login` accepting `LoginRequest` body: verify password via `verify_password(request.password, config.password_hash)`, if valid create token via `get_token_service().create_token()`, set cookie `session` with `httponly=True, samesite="lax", path="/"` (no max_age = session cookie, no secure=True for HTTP LAN). Return `{"status": "ok"}`. If invalid, raise HTTPException(status_code=401, detail="Invalid password").
   - `POST /logout` endpoint: clear the session cookie by setting max_age=0. Return `{"status": "ok"}`.

5. Update `server/app/routers/server_info.py`: add `read_only`, `receive`, `password_required` (derived from `config.password_hash is not None`), `hostname` (from `socket.gethostname()`) to the ServerInfo response.

6. Update `server/app/main.py` `create_app()`:
   - Import and include auth router
   - Conditionally add AuthMiddleware: `if config.password_hash is not None: application.add_middleware(AuthMiddleware, token_service=get_token_service())`
   - The middleware must be added AFTER CORSMiddleware (Starlette processes middleware in reverse order, so AuthMiddleware added second runs first)

7. Update `server/app/routers/websocket.py` `websocket_endpoint`: At the top of the function, before `manager.connect()`, check if password is enabled (`config.password_hash is not None`). If so, parse the cookie from websocket.headers, validate via token_service. If invalid, `await websocket.close(code=4001)` and return.

Note: conftest.py fixtures for password, read-only, and receive modes were already created in Plan 01. Use `async_client_with_password` and related fixtures directly in tests.

Write integration tests in `server/tests/test_auth.py`.
