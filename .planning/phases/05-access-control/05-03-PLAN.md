1. Update `client/src/types/serverInfo.ts` -- extend `ServerInfo` with `read_only: boolean`, `receive: boolean`, `password_required: boolean`, `hostname: string`.

2. Create `client/src/api/auth.ts`:
   - `export async function login(password: string): Promise<void>` -- POST to `/api/auth/login` with `{ password }`. On 401, throw ApiError. On 200, cookie is automatically set by browser (same-origin).
   - `export async function logout(): Promise<void>` -- POST to `/api/auth/logout`.

3. Create `client/src/components/LoginPage.tsx`:
   - Full-page centered form with: title "WiFi File Server", subtitle "This server is password protected", password input field (type="password", autoFocus), submit button "Sign In", error message area.
   - State: password (string), error (string | null), isSubmitting (boolean).
   - On submit: call `login(password)`. On success, call `onLoginSuccess()` callback prop. On 401, set error "Invalid password". On other errors, set error "Connection failed".
   - Clear error on password change.
   - Styling: Tailwind, dark mode support, centered vertically and horizontally. Clean, minimal design consistent with app style (gray-50 bg, dark:gray-900, rounded card with shadow).
   - Props: `onLoginSuccess: () => void`.

4. Update `client/src/main.tsx` to become the mode-aware root:
   - Create a `Root` component that:
     a. Fetches server-info on mount to get `read_only`, `receive`, `password_required`, `hostname`
     b. Stores `serverMode` state (null = loading, then the mode object)
     c. Stores `isAuthenticated` state (starts false, set true on login success or if password not required)
     d. If loading: show minimal loading spinner (centered)
     e. If `password_required && !isAuthenticated`: render `<LoginPage onLoginSuccess={() => setIsAuthenticated(true)} />`
     f. If `receive`: render `<DropBoxPage hostname={serverMode.hostname} />` (created in Task 2)
     g. Otherwise: render `<App serverMode={serverMode} />`
   - Handle 401 from server-info fetch: if password_required, show login. Use a try/catch around fetchServerInfo -- if 401, we know password is required but we are not authenticated.
   - Actually, /api/server-info is exempt from auth middleware, so it always returns. Check `password_required` field instead. But API calls TO other endpoints will get 401 if unauthenticated. So: fetch server-info (always works), if password_required, show login first. After login, re-render App.

5. Define `ServerMode` type in a new file or inline in main.tsx:
   ```typescript

     readOnly: boolean;
     receive: boolean;
     passwordRequired: boolean;
     hostname: string;
   }
   ```
