  <behavior>
    Stream lifecycle:
    - open_stream(request_id) creates a StreamState with bounded asyncio.Queue(maxsize=64)
    - open_stream with duplicate request_id raises TunnelError
    - open_stream when at MAX_STREAMS (100) raises StreamLimitError
    - Dispatching DATA frame to open stream puts payload in that stream's queue
    - Dispatching DATA frame to non-existent stream raises StreamNotFoundError
    - CLOSE frame sets stream.closed event and removes stream from tracking dict
    - CANCEL frame sets stream.closed event and removes stream from tracking dict
    - ERROR frame sets stream.closed event and removes stream from tracking dict
    - After CLOSE/CANCEL/ERROR, stream is fully cleaned up (not in _streams dict)

    Multiplexing:
    - Two concurrent streams receive their own data without cross-contamination
    - Stream A receiving 3 DATA frames and Stream B receiving 2 DATA frames — each queue has correct count
    - CANCEL on Stream A does not affect Stream B

    Sending:
    - send_data(request_id, payload) serializes and sends binary frame via ws.send_bytes
    - send_open(request_id, metadata_dict) serializes OPEN frame with JSON metadata as payload, sends binary
    - send_close(request_id) serializes CLOSE frame with empty payload, sends binary
    - send_cancel(request_id) serializes CANCEL frame with empty payload, sends binary

    Control messages:
    - send_control({"type": "ping"}) sends JSON text frame via ws.send_text
    - send_control without "type" key raises TunnelError
    - Receiving text frame dispatches to control handler (verified via MockWebSocket)
    - Receiving binary frame dispatches to frame handler (deserialized and routed by request_id)
  </behavior>

<task type="auto" tdd="true">
  <name>Task 2: Backpressure, heartbeat, and first-byte timeout with TDD</name>
  <files>

  <behavior>
    Backpressure:
    - Filling queue to capacity (64 frames) and adding one more blocks the put() call
    - After consumer reads one frame from full queue, the blocked put() completes
    - No frames are dropped — all 64+1 frames are eventually received by consumer
    - Backpressure is per-stream — Stream A being full does not block Stream B

    Heartbeat:
    - start_heartbeat() creates a background asyncio.Task that sends ping control messages every HEARTBEAT_INTERVAL_S
    - When pong is received within the interval, missed counter resets to 0
    - After HEARTBEAT_MISSED_LIMIT (3) consecutive missed pongs, connection tears down
    - Teardown closes all open streams (sets closed event on each)
    - stop() cancels the heartbeat task without "Task was destroyed" warnings
    - handle_pong() method signals the pong event

    First-byte timeout:
    - read_stream(request_id, timeout_s) awaits first item from stream queue with timeout
    - If no data arrives within timeout_s, raises FirstByteTimeoutError
    - Once first byte arrives, subsequent reads have no timeout (stream continues indefinitely)
    - read_stream on a closed stream (closed event set, empty queue) returns sentinel or raises appropriately

    Teardown:
    - close() cancels heartbeat task, closes all streams, is idempotent (calling twice is safe)
  </behavior>
