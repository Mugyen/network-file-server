<task type="auto" tdd="true">
  <name>Task 1: Schemas, exceptions, and service functions for all file operations</name>
  <files>
    server/app/models/schemas.py,

    server/app/exceptions.py,
    server/app/services/file_service.py,

  <behavior>
    - upload_file(base_dir, relative_dir, upload_file, conflict_resolution) saves file to disk via aiofiles chunked write (1MB chunks), returns UploadResult with name, size, size_display
    - upload_file raises FileConflictError when file exists and conflict_resolution is not provided
    - upload_file with OVERWRITE writes to temp file then os.replace() for atomicity
    - upload_file with RENAME appends _1, _2 etc to filename stem
    - upload_file with SKIP returns UploadResult with skipped=true
    - download_file(base_dir, relative_path) returns resolved Path after validating it is a file (not directory)
    - download_file raises ValueError if path points to a directory
    - delete_paths(base_dir, paths) deletes each path -- os.remove for files, shutil.rmtree for directories
    - delete_paths raises PathTraversalError for any invalid path, FileNotFoundError for missing
    - rename_path(base_dir, old_relative, new_name) renames file/dir, returns new relative path
    - rename_path raises InvalidFileNameError for empty names, names containing /, .., or null bytes
    - rename_path raises FileConflictError if target name already exists
    - create_folder(base_dir, parent_relative, folder_name) creates directory, returns new relative path
    - create_folder raises InvalidFileNameError for invalid names
    - create_folder raises FileConflictError if folder already exists
    - download_as_zip(base_dir, paths) returns a ZipStream generator for StreamingResponse
    - All functions call resolve_safe_path as first operation on user-provided paths
  </behavior>
  <action>
    1. Install zipstream-ng: `uv add zipstream-ng`
    2. Add to server/app/models/enums.py: ConflictResolution enum with OVERWRITE, RENAME, SKIP values
    3. Add to server/app/models/schemas.py: UploadResult(name: str, size: int, size_display: str, skipped: bool), RenameRequest(path: str, new_name: str), DeleteRequest(paths: list[str]), CreateFolderRequest(parent_path: str, name: str), DownloadZipRequest(paths: list[str])
    4. Add to server/app/exceptions.py: FileConflictError(path: str, existing_path: str) and InvalidFileNameError(name: str, reason: str)
    5. Add to server/app/services/file_service.py the 6 service functions listed in behavior. Use aiofiles for upload_file (async chunked write with UPLOAD_CHUNK_SIZE = 1024 * 1024). Use zipstream.ZipStream for download_as_zip. All functions MUST call resolve_safe_path before any filesystem operation. Use os.replace() for atomic overwrite. Use shutil.rmtree for directory deletion. Validate new names reject empty, /, .., and null bytes.
    6. Write tests FIRST (RED), then implement (GREEN). Extend server/tests/test_file_service.py with tests for all new functions. Use tmp_path fixture for isolated filesystem tests. Include path traversal attempt tests for every function.
  </action>
