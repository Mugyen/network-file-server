    1. Create client/src/types/upload.ts:
       - UploadStatus as const: QUEUED, UPLOADING, DONE, FAILED, CONFLICT (use `as const` pattern, not TypeScript enum keyword)
       - ConflictAction as const: OVERWRITE, RENAME, SKIP
       - UploadFileState: { id: string, file: File, status: UploadStatus, progress: number, error: string | null, conflictAction: ConflictAction | null }
       - UploadResult: { name: string, size: number, size_display: string, skipped: boolean }

    2. Extend client/src/api/client.ts:
       - Add uploadWithProgress(file: File, targetPath: string, conflictResolution: string | null, onProgress: (percent: number) => void): Promise<UploadResult>
       - Use XMLHttpRequest (NOT fetch) per research. Attach to xhr.upload.addEventListener("progress", ...) for upload progress tracking.
       - Build FormData with file appended as "files" key
       - URL: /api/files/upload?path={encodeURIComponent(targetPath)} plus &conflict_resolution={value} if conflictResolution is not null
       - Resolve promise on xhr.load with status 2xx, reject with ApiError on error
       - On 409 status, throw a typed error that the caller can detect as a conflict

    3. Extend client/src/api/files.ts:
       - Add uploadFiles function if needed as a higher-level wrapper (or keep it in the hook)

    4. Create client/src/hooks/useDragDrop.ts:
       - Export function useDragDrop(onDrop: (files: FileList) => void) returning { isDragging: boolean, dragHandlers: { onDragEnter, onDragLeave, onDragOver, onDrop } }
       - Use drag counter pattern from 02-RESEARCH.md to prevent overlay flicker from child element events
       - Increment counter on dragenter, decrement on dragleave, reset to 0 on drop
       - isDragging = counter > 0
       - onDrop: preventDefault, reset counter, call onDrop callback with e.dataTransfer.files

    5. Create client/src/hooks/useUpload.ts:
       - Export function useUpload(currentPath: string, onUploadComplete: () => void) returning { uploadFiles: (files: FileList) => void, uploads: UploadFileState[], pendingConflict: UploadFileState | null, resolveConflict: (action: ConflictAction) => void, clearCompleted: () => void, retryFailed: (id: string) => void, isUploading: boolean, collapsed: boolean, toggleCollapsed: () => void }
       - Manage uploads array in state with unique IDs (crypto.randomUUID())
       - Concurrency limit: 3 simultaneous uploads, queue the rest
       - For each upload: set status UPLOADING, call uploadWithProgress, update progress on callback, set DONE on success, set FAILED on error
       - On 409 conflict: set status CONFLICT, store as pendingConflict. When user calls resolveConflict(action), retry the upload with conflict_resolution query param.
       - retryFailed: reset status to QUEUED, re-queue
       - clearCompleted: remove DONE and SKIPPED entries from array
       - onUploadComplete callback fires after each successful upload (used by App to refresh file list)
       - collapsed state for panel toggle
  </action>
